# Лимиты и пороги оптимизации (Thresholds)

Для поддержания высокого качества ответов и минимального расхода токенов, файлы всегда-загружаемого контекста должны стремиться к следующим показателям:

| Файл | Оптимально (B) | Критично (B) | Содержание |
|---|---|---|---|
| `AGENTS.md` | ~2,500 | 8,000 | Только "жесткие" правила дисциплины |
| `USER.md` | ~1,200 | 3,000 | Текущий профиль и главные цели |
| `SOUL.md` | ~1,000 | 2,500 | Принципы личности и границы |
| `MEMORY.md` | ~1,500 | 5,000 | Ссылки на доки и ключевые факты |
| `HEARTBEAT.md` | ~800 | 2,000 | Короткий чеклист действий |
| `TOOLS.md` | ~300 | 1,000 | Локальные спецификации |

## Правило 20К
Если суммарный вес этих 6 файлов превышает **20,000 байт**, ассистент обязан предложить аудит оптимизации в следующем сообщении.

## Приоритет выноса
1. Гайды "Как делать..." → в `/docs/protocols/`.
2. Технические параметры (IP, ID) → в `/docs/infra/`.
3. История событий и решений → в `/memory/archive/`.
