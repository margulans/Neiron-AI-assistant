## Алгоритм обработки

### Шаг 1 — Классификация

Определи уровень сообщения по `references/rules.md`:
- 🟢 Уровень 1 — рутинные данные (вес, давление, приём лекарства)
- 🟡 Уровень 2 — важные данные (анализы, визит к врачу, новое лекарство)
- 🔴 Уровень 3 — требует апрува (встречи, задачи, напоминания, действия от имени Маргулана)
- 🚨 Экстренный — острые симптомы, «плохо», скорая

### Шаг 2 — Выполнение

**🟢 Уровень 1:**
1. Сохрани данные в нужный файл (Авиценна, memory/YYYY-MM-DD.md)
2. Ответь Айганым: `✅ Записано: [суть]`
3. Не уведомляй Маргулана

**🟡 Уровень 2:**
1. Сохрани данные в нужный файл
2. Ответь Айганым: `✅ Записано: [суть]`
3. Уведоми Маргулана (telegram:685668909): `✅ Айганым передала: [суть]. Сохранено в [файл].`

**🔴 Уровень 3:**
1. НЕ выполняй действие
2. Добавь в sandbox: `memory/sandbox/YYYY-MM-DD.md` (см. формат ниже)
3. Ответь Айганым: `⏳ Передано Маргулану на согласование`
4. Немедленно отправь PR Маргулану (telegram:685668909) — см. формат PR

**🚨 Экстренный:**
1. Немедленно уведоми Маргулана: `🚨 СРОЧНО от Айганым: [суть дословно]`
2. Ответь Айганым: `✅ Маргулан уведомлён`

## Формат sandbox файла

Путь: `memory/sandbox/YYYY-MM-DD.md`

```markdown
# Sandbox | YYYY-MM-DD

## Ожидают апрув

### #N | HH:MM
- **Тип:** [Календарь / Задача / Напоминание / Другое]
- **Данные:** [что именно попросила Айганым]
- **Статус:** pending
```

Если файл уже существует — добавляй новые записи в конец раздела "Ожидают апрув".

## Обработка ответа Маргулана на PR

Когда Маргулан отвечает на PR:

**«апрув»** → выполни все pending пункты из sandbox, очисти sandbox
**«апрув 1,3»** → выполни только указанные пункты, остальные удали
**«отклонить 2»** → удали пункт #2 из sandbox без выполнения
**«изменить 1: ...»** → измени данные пункта #1 и выполни

После обработки:
- Уведоми Айганым о результате: `✅ Маргулан одобрил: [суть]` или `❌ Отклонено: [суть]`
- Обнови/удали sandbox файл
- Выполни git commit
